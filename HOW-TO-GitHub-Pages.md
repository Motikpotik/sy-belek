# 🚀 Установка SY Hotels Belek на iPhone через GitHub Pages

**Твоя ситуация:** GitHub-аккаунт есть, платить нельзя, через iOS Files локально не работает.

**Решение:** GitHub Pages — бесплатно навсегда, реальный URL, работает офлайн.

---

## ⏱️ Время: 10-15 минут

## 📋 Что нужно

1. ✅ GitHub-аккаунт (есть)
2. ✅ iPhone/iPad с Safari
3. ✅ ZIP-файл `sy-belek.zip` (я его подготовил)
4. ❌ ПК НЕ нужен

---

## 🎯 План

1. Распаковать ZIP на iPhone
2. Залить файлы в новый GitHub-репозиторий через мобильный Safari
3. Включить GitHub Pages
4. Открыть URL в Safari → Add to Home Screen
5. Готово — PWA с офлайн-режимом

## 📥 Шаг 1. Скачать и распаковать ZIP

### 1.1 Скачать ZIP
Я дам тебе ссылку на `sy-belek.zip`. Нажми на неё в Safari → файл скачается в **Files → Downloads**.

### 1.2 Распаковать
1. Открой приложение **Files** (Файлы)
2. Зайди в **Downloads** (Загрузки)
3. Найди `sy-belek.zip`
4. **Долгое нажатие** на файл → **«Распаковать»** (Uncompress)
5. Появится папка `sy-belek` с файлами:
   ```
   sy-belek/
   ├── index.html
   ├── manifest.webmanifest
   ├── sw.js
   ├── HOW-TO-INSTALL-iPhone.md
   ├── HOW-TO-INSTALL-details.md
   └── icons/
       ├── apple-touch-icon.png
       ├── apple-splash-2048.png
       ├── icon-192.png
       ├── icon-512.png
       ├── favicon.svg
       ├── favicon-32.png
       └── favicon-16.png
   ```

✅ **Готово — файлы на iPhone.**

## 🐙 Шаг 2. Создать репозиторий на GitHub

### 2.1 Открой GitHub в Safari
Перейди на **https://github.com/new** (войдёшь в аккаунт если не вошёл)

### 2.2 Создай репозиторий
- **Repository name:** `sy-belek` (именно так, маленькими буквами, с дефисом)
- **Description:** `SY Hotels Belek — обзор эксперта` (необязательно)
- **Visibility:** ✅ **Public** (важно — для бесплатного GitHub Pages)
- **Initialize this repository with:** ❌ НЕ ставь галочки (Add README, .gitignore, license)
- Нажми зелёную кнопку **«Create repository»**

### 2.3 Загрузи файлы
Теперь ты на странице пустого репозитория. Там есть ссылка **«uploading an existing file»** — нажми её.

⚠️ **Важно:** мобильный GitHub не поддерживает drag-and-drop папок. Поэтому:

1. Нажми **«choose your files»**
2. Откроется iOS Files picker
3. Зайди в **Files → Downloads → sy-belek/**
4. **Выдели ВСЕ файлы** (долгое нажатие на одном → «Select» → отметь все):
   - `index.html`
   - `manifest.webmanifest`
   - `sw.js`
   - `HOW-TO-INSTALL-iPhone.md`
   - `HOW-TO-INSTALL-details.md`
   - Все 7 файлов из папки `icons/`
5. Нажми **«Open»** / **«Добавить»**
6. GitHub покажет список загруженных файлов
7. Внизу нажми зелёную кнопку **«Commit changes»**

⚠️ **Проблема:** GitHub может не дать загрузить папку `icons/` целиком — только отдельные файлы в корень.

### 2.4 Если папка icons не загрузилась

Если `icons/` не создалась автоматически (файлы легли в корень), нужно создать папку вручную:

1. На странице репозитория нажми **«Add file» → «Create new file»**
2. В поле имени введи: `icons/.gitkeep` (это создаст пустую папку `icons`)
3. Нажми **«Commit new file»**
4. Теперь снова **«Add file» → «Upload files»**
5. Загрузи каждый файл из `icons/` по очереди
6. GitHub сам положит их в корень — **придётся переименовать** через веб-интерфейс:
   - Открой файл → нажми ✏️ (Edit) → переименуй, добавив `icons/` перед именем

⚠️ **Это сложно на мобильном.** Альтернатива — использовать приложение **Working Copy** (есть бесплатная версия):

1. Скачай **Working Copy** из App Store
2. Открой → клонируй свой репозиторий `sy-belek`
3. Через iOS Files скопируй ВСЮ папку `sy-belek/` (с подпапкой `icons/`) в клонированный репозиторий
4. В Working Copy нажми **Commit** → **Push**
5. Готово — все файлы с правильной структурой в GitHub

## 🌐 Шаг 3. Включить GitHub Pages

1. Открой свой репозиторий `https://github.com/твойлогин/sy-belek`
2. Нажми вкладку **«Settings»** (Настройки, сверху справа)
3. Слева в меню найди **«Pages»** (раздел Code & automation)
4. В разделе **«Build and deployment»**:
   - **Source:** `Deploy from a branch`
   - **Branch:** выбери `main` (или `master`) → папка `/ (root)`
   - Нажми **«Save»**
5. ⏳ Подожди 1-2 минуты (наверху появится надпись «Your site is live at...»)
6. **Скопируй URL** вида: `https://твойлогин.github.io/sy-belek/`

✅ **Готово — сайт в интернете!**

## 📱 Шаг 4. Установить как приложение на iPhone

1. Открой Safari на iPhone
2. Вставь скопированный URL: `https://твойлогин.github.io/sy-belek/`
3. Нажми **Go** → сайт загрузится (золотая иконка SY в вкладке, всё красиво)
4. Нажми кнопку **Share** (квадрат со стрелкой ↑ вверх, внизу по центру)
5. Прокрути список вниз → **«Добавить на главный экран»** (Add to Home Screen)
6. Подтверди название (по умолчанию «SY Hotels Belek»)
7. Нажми **«Добавить»** (верхний правый угол)

✅ **На рабочем столе появилась золотая иконка SY!**

## 🧪 Шаг 5. Проверить, что всё работает

### Проверка 1: запускается как приложение
- Нажми на иконку SY на рабочем столе
- ✅ Должен запуститься без адресной строки Safari (полноэкранный режим)
- ✅ Splash screen — золотой круг с буквами SY на тёмном фоне

### Проверка 2: работают вкладки и счётчики
- Открой «Обзор» — счётчики (8.7, 4.3/5, 240, 14) должны анимированно «отщёлкнуться» от 0
- Перейди по вкладкам: Номера → Горки → Еда → Бассейны → SPA → Сравнение
- Все фото должны грузиться с syhotels.com

### Проверка 3: офлайн-режим
- Открой приложение, пройди по всем вкладкам (фото загрузятся и закэшируются)
- Включи **Авиарежим** (настройки → авиарежим, или Control Center)
- Снова открой приложение → должно работать офлайн (фото из кэша)
- ✅ Если работает — Service Worker активен, ты красавчик

### Проверка 4: кнопки «Поделиться»
- Прокрути в самый низ → секция «📤 Поделиться обзором»
- Нажми «Telegram» → откроется окно Telegram с готовым текстом
- Нажми «Копировать» → всплывёт toast «Ссылка скопирована ✓»

### Проверка 5: Schema.org
- Открой **https://search.google.com/test/rich-results**
- Вставь свой URL `https://твойлогин.github.io/sy-belek/`
- Нажми **«Test URL»** → Google должен распознать Hotel + AggregateRating + FAQ
- Это значит — в поиске Google рядом с твоей страницей будут звёзды рейтинга

## 🆘 Если что-то не работает

### Проблема: сайт не открывается по URL
- Подожди ещё 2-3 минуты (GitHub Pages иногда тупит)
- Проверь: Settings → Pages → должен быть зелёный бейдж «Live»
- Проверь: репозиторий Public (не Private)
- Проверь: branch = main, folder = /root

### Проблема: иконки не отображаются (битые картинки в вкладке)
- Проверь, что папка `icons/` загрузилась с правильной структурой
- Открой `https://github.com/твойлогин/sy-belek` — должна быть папка `icons/` с 7 файлами внутри
- Если файлы легли в корень без папки — используй Working Copy (см. шаг 2.4)

### Проблема: фото отеля не грузятся
- Это значит нет интернета, или syhotels.com блокирует
- Сайт всё равно откроется, но фото будут placeholder'ами
- Service Worker кэширует фото при первой загрузке — потом будут работать офлайн

### Проблема: PWA не устанавливается (нет Add to Home Screen)
- Должно быть открыто в **Safari** (не Chrome!)
- Должен быть HTTPS (GitHub Pages даёт HTTPS автоматически)
- Manifest должен быть доступен по URL: `https://твойлогин.github.io/sy-belek/manifest.webmanifest`
- Service Worker по URL: `https://твойлогин.github.io/sy-belek/sw.js`

### Проблема: Service Worker не регистрируется
- Открой Safari → Develop → [твой iPhone] → [сайт]
- (На iOS без Mac это недоступно — просто проверь, что `sw.js` доступен по URL)
- Если файл есть, но не регистрируется — проверь, что он на том же домене

## 🔄 Как обновлять контент

Если захочешь что-то поменять в HTML:

1. Открой `https://github.com/твойлогин/sy-belek/blob/main/index.html`
2. Нажми ✏️ (Edit) в правом верхнем углу
3. Внеси изменения
4. **Commit changes**
5. Через 1-2 минуты GitHub Pages автоматически обновится
6. При следующем открытии приложения Service Worker подтянет новую версию
